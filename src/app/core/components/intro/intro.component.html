Introduction check This application is developed using Angular. Please note the key points related to architecture of the app that I used. check App is modularized. All the modules are directly under the folder 'src/app'. Components are inside modules. check Core module has three components intro, header and sidenav-list which are used only once in the app. check Shared module has single loader component. Components in shared module can be reused throughout the app in any module. check Angular material is extensively used. All the material modules used in app are imported into a module names 'MaterialModule'. This module is imported whereever necessary. check My Info data is loaded from a memory api. Once data is fetched it is saved in a service ('MyInfoService') so that next time user loads the component another HTTP request is not made. check Reactive form is used to handle data. Validations are added. Name is mandatory and age cannot be less than 0. Email and mobile number format is also validated. For mobile number validation I used a simple regex. Please add mobile number without spaces and hyphens to test validation. If there are validation errors appropriate error messages are show. Once the form is submitted, the edited data is saved in the service 'MyInfoService'. check Navigation bar is responsive. Please try to reduce the width of screen below 600px and notice that nav bar will collapse. check Used 'ngx-toastr' npm module to show toast messages when a user action completes or error occurs. check Run ng lint to find any linting errors. check Please checkout to master branch.
